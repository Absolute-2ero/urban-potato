## 2. 核心交互流程

### 2.1 搜索主流程

```
用户在首页搜索框输入
    │
    ├─ 输入 ≥ 2 字符 → Autocomplete 下拉展示（防抖 200ms）
    │   ├─ 用户点击建议词 → 填入搜索框，自动执行搜索
    │   └─ 用户继续输入 → 下拉实时刷新
    │
    └─ 用户按 Enter 或点击搜索按钮
        → 跳转 /search?q=...
        → 搜索结果页加载（Skeleton 占位，≤500ms 后真实内容）
        │
        ├─ 左侧 Facet 面板：
        │   ├─ 勾选 / 取消 Facet 标签 → URL 参数更新，结果区重新加载
        │   └─ 已选标签以 Tag 形式显示在搜索框下方，支持 × 单独移除
        │
        ├─ 顶部排序切换：综合 / 饮食匹配 / 评分 / 距离
        │   → 切换排序模式 → 结果区重排（无需重搜，前端重排）
        │
        └─ 点击结果卡片 → 跳转 /restaurant/:id
```

### 2.2 食物搜索与日志录入流程

```
用户进入 /diet/log
    → 点击"+ 添加食物"按钮
    → 搜索输入框弹出（Modal 或底部 Drawer）
        │
        用户输入食物名
        │
        ├─ 数据库命中 → 展示食物列表（含热量/蛋白质预览）
        │   └─ 用户选择食物 → 进入分量输入步骤
        │
        └─ 数据库未命中
            → 展示"数据库中没有该食物，正在通过 AI 查询..."（Loading）
            → LLM 返回结果
            ├─ 成功 → 展示估算数据 + 确认弹窗
            │   └─ 用户点击"确认并使用" → 数据入库 → 进入分量输入步骤
            └─ 失败 → 展示"查询失败，请手动输入或稍后重试"

分量输入步骤：
    → 展示食物名 + 单位（克）输入框 + 餐次选择（早/午/晚/零食）
    → 实时计算热量预览（"本次摄入约 XXX kcal"）
    → 用户点击"添加" → POST /api/diet/log → 日志列表刷新
```

### 2.3 饮食档案设置流程

```
用户进入 /diet/profile
    → 展示当前档案（空档案展示引导提示）
    │
    饮食类型区（多选 Checkbox Group）：
        □ 纯素 Vegan        □ 素食 Vegetarian
        □ 清真 Halal        □ 无麸质 Gluten-Free
        ... （15 种，可折叠显示全部）
    │
    过敏原区（多选，含常见过敏原快选 + 自定义输入）：
        □ 花生  □ 坚果  □ 贝壳类  □ 大豆  □ 麸质
        [+ 添加其他过敏原]
    │
    每日目标区（数字输入，可选填）：
        热量目标：[____] kcal
        蛋白质：[____] g   脂肪：[____] g   碳水：[____] g
    │
    点击"保存档案" → PUT /api/diet/profile
        → 成功 Toast："档案已更新，搜索结果将基于你的饮食需求个性化"
```

## 3. 页面交互规范

### 3.1 首页（/）

**核心元素：**
- Hero 区：大标题 + 副标题 + 居中搜索框（Hero 插图来自 frontend-illustration-generator）
- 搜索框下方：常用饮食类型快选标签（点击直接填入搜索框并执行搜索）
- 功能说明区：三栏图文（"记录饮食" / "精准搜索" / "匹配推荐"）
- 未登录时：Header 右侧显示"登录 / 注册"按钮

**交互细节：**
- 搜索框自动 focus（页面加载后 300ms）
- 常用标签 hover 时：颜色加深 + 轻微上移（translateY -2px）
- 首次访问（无 Session）：不弹登录要求，匿名可直接搜索

### 3.2 搜索结果页（/search）

**状态管理：**
- URL 参数为唯一真实来源：`?q=素食&diet_labels=vegan&sort=default&page=1`
- 浏览器前进后退正常工作（URL 参数驱动）

**加载状态：**
- 首次加载：结果区显示 Skeleton（3 个卡片骨架）
- Facet 切换：结果区显示半透明遮罩 + Spinner（不清空已有结果）
- Autocomplete：防抖 200ms，下拉最多 8 条

**空结果状态：**
- 展示空状态插图（无结果场景）
- 提供"建议"区：
  - "试试去掉 [diet_labels 过滤]"（点击一键移除）
  - "附近城市的类似餐厅"（降级展示，若有数据）

**相关性解释展开：**
- 每个结果卡片底部有折叠区域"为什么推荐这家？"
- 展开后显示：
  - BM25 命中词：`"素食" 命中菜品名 × 3，命中描述 × 1`
  - 饮食标签匹配：`✅ vegan (餐厅级确认)`
  - 如有冲突：`⚠️ 含你的过敏原：花生`

### 3.3 餐厅详情页（/restaurant/:id）

**布局：**
- 左栏（60%）：餐厅基本信息 + 菜品列表
- 右栏（40%）：地图（iframe 嵌入）+ 营业时间 + 收藏按钮

**菜品列表交互：**
- 默认按饮食标签匹配度排序（用户档案中的标签命中的菜品排前）
- 每道菜品展示：名称 + 价格 + 饮食标签 + 热量（若有）
- 点击菜品展开营养详情（蛋白质/脂肪/碳水）

**收藏按钮：**
- 未登录时：点击 → 弹出"请先登录以使用收藏功能" Toast
- 已登录未收藏：书签空心图标，点击 → 变为实心 + "已收藏"动画
- 已登录已收藏：实心图标，点击 → 确认取消收藏

**相关性反馈：**
- 页面底部卡片："这个结果对你有帮助吗？"
- [👍 有帮助]  [👎 没有帮助]
- 点击后：按钮变为已选态，POST /api/feedback，Toast "感谢反馈"

### 3.4 饮食日志页（/diet/log）

**日期切换：**
- 顶部日期栏：← [今天 2026-05-22] →（左右切换日期）
- 点击日期文字 → 日历选择器弹出（Ant Design DatePicker）

**餐次分组展示：**
```
早餐 ─────────────────────────── 总计 350 kcal
  □ 燕麦粥 200g        136 kcal    [✏️ 编辑] [×]
  □ 蓝莓 100g          57 kcal     [✏️ 编辑] [×]

午餐 ─────────────────────────── 总计 720 kcal
  □ 红烧肉 100g        395 kcal    [✏️ 编辑] [×]
  ...

[+ 添加食物]
```

**底部今日汇总：**
- 固定在页面底部（sticky）
- 展示：热量进度条 + 三大营养素进度条
- 点击汇总区 → 展开详细数据（完整营养数据表）

### 3.5 通用交互规范

**Toast 提示：**
- 成功：绿色，自动消失 3s
- 错误：红色，需手动关闭
- 位置：页面顶部居中

**Modal 弹窗：**
- 最大宽度：560px
- 背景遮罩：`rgba(0,0,0,0.45)`
- 点击遮罩关闭（非危险操作）
- 危险操作（删除日志）：使用 Ant Design `Modal.confirm`

**表单验证：**
- 实时校验（onChange，非 onSubmit）
- 错误信息在字段下方展示，红色，12px

**加载反馈：**
- 按钮操作：按钮显示 `loading` prop（防重复提交）
- 页面加载：Skeleton 组件占位
- 列表加载更多：底部 Spin

## 4. 键盘交互

| 快捷键 | 位置 | 动作 |
|--------|------|------|
| `/` | 全局 | 聚焦到搜索框 |
| `Esc` | 搜索框 focus 时 | 清空搜索框 / 关闭 Autocomplete |
| `↑ ↓` | Autocomplete 下拉 | 上下导航建议项 |
| `Enter` | Autocomplete 下拉 | 选中当前高亮的建议项 |
| `Tab` | 表单 | 按序聚焦下一个输入项 |
