# 测试策略

**版本：** v1.0
**更新日期：** 2026-05-22

---

## 1. 测试目标

| 目标 | 说明 |
|------|------|
| 核心业务不回归 | 食物搜索（含 LLM fallback）、餐厅搜索、饮食日志录入三条主链路每次发布前必须通过 |
| IR 检索质量 | 评估指标（Precision@10、NDCG@10）在每次迭代后不低于基线值 |
| 过敏原安全 | 过敏原警告逻辑不能有遗漏，BR-08 的覆盖测试为高优先级 |
| LLM 数据安全 | LLM 返回的脏数据（calories=-1、JSON 解析失败）不进入数据库 |
| 排序稳定性 | 权重配置变更后，排序结果变化方向符合预期 |

## 3. 单元测试

### 3.1 覆盖范围

| 模块 | 测试文件 | 测试要点 |
|------|---------|---------|
| `ir/query_parser.py` | `tests/test_query_parser.py` | 分词正确性、同义词扩展、意图识别 |
| `ir/synonyms.py` | `tests/test_synonyms.py` | 词典加载、同义词命中、边界词 |
| `ir/spell_checker.py` | `tests/test_spell_checker.py` | 编辑距离计算、纠错词正确性 |
| `services/ranking_service.py` | `tests/test_ranking_service.py` | 评分融合权重、过敏原惩罚、排序方向 |
| `services/food_service.py` | `tests/test_food_service.py` | LLM 响应解析、calories=-1 处理、JSON 解析失败 |
| `crawler/nlp_labeler.py` | `tests/test_nlp_labeler.py` | 标签推断规则正确性 |
| `crawler/pipeline.py` | `tests/test_pipeline.py` | 去重逻辑（坐标近邻）、字段标准化 |

### 3.2 测试原则

- **不 mock 数据库**：使用测试用 PostgreSQL Schema（`test_dietsearch`）和测试 SQLite 文件
- 每个测试函数前执行 fixture 清理（`truncate` 或事务回滚）
- **ES 测试**：使用 `elasticsearch-test-utils` 的内存 ES 或独立测试索引（`test_restaurants`）
- **LLM 测试**：使用 `httpx-mock` 拦截 LLM API 调用，返回固定 fixture 响应

### 3.3 运行命令

```bash
cd backend
pytest tests/ -v --tb=short                  # 全部单元测试
pytest tests/test_ranking_service.py -v      # 仅测试排序模块
pytest tests/ -v --cov=services --cov-report=term-missing  # 覆盖率报告
```

## 5. IR 系统评估（离线）

这是本项目特有的测试层，用于评估检索质量，独立于功能测试运行。

### 5.1 评估数据集

**查询集构建：**

| 查询类型 | 数量 | 来源 |
|---------|------|------|
| 饮食类型查询（"vegan restaurants"、"清真餐厅"） | 30 条 | 人工撰写 |
| 营养目标查询（"高蛋白低脂肪"、"keto friendly"） | 20 条 | 人工撰写 |
| 混合查询（"素食 日式 性价比高"） | 20 条 | 人工撰写 |
| 过敏原排除查询（"无花生 中式"） | 10 条 | 人工撰写 |
| 拼写错误查询（"vegann"、"gluten fre"） | 10 条 | 构造 |

**相关性标注：**
- 每条查询对应 10+ 个餐厅文档，由 2 名标注者独立标注相关性（0/1/2 三级）
- 标注不一致时取众数，差异 > 1 时讨论决定
- 相关性文件格式：`data/eval/qrels.tsv`（`query_id \t restaurant_id \t relevance`）

### 5.2 评估指标

| 指标 | 计算方式 | v1.0 基线目标 |
|------|---------|--------------|
| Precision@10 | Top-10 结果中相关文档比例 | ≥ 0.75 |
| Recall@10 | Top-10 中找到的相关文档 / 总相关文档 | ≥ 0.60 |
| NDCG@10 | 归一化折损累计增益（考虑排名位置） | ≥ 0.70 |
| MRR | 平均倒数排名（第一个相关文档的排名） | ≥ 0.80 |
| 过敏原警告覆盖率 | 含过敏原餐厅展示警告的比例 | 100%（零容忍） |

### 5.3 评估运行

```bash
# 运行 IR 评估
python -m ir.evaluate \
    --queries data/eval/queries.json \
    --qrels data/eval/qrels.tsv \
    --output reports/eval_$(date +%Y%m%d).json

# 输出格式
{
  "timestamp": "2026-05-22T10:00:00",
  "metrics": {
    "precision_at_10": 0.78,
    "recall_at_10": 0.63,
    "ndcg_at_10": 0.74,
    "mrr": 0.85,
    "allergen_warning_coverage": 1.0
  },
  "per_query_breakdown": [ ... ]
}
```

### 5.4 评估触发时机

| 时机 | 动作 |
|------|------|
| 修改排序权重（ranking.yaml）| 必须运行评估，对比前后指标变化 |
| 修改同义词词典 | 必须运行评估 |
| 修改饮食匹配评分公式 | 必须运行评估 |
| 新增餐厅数据批次 | 建议运行评估 |
| v1.x 版本发布前 | 必须运行，指标不得低于基线 |

## 7. 数据安全测试

| 测试项 | 验证内容 |
|--------|---------|
| 跨用户数据隔离 | 用户 A 无法读取用户 B 的日志（BR-02） |
| LLM 脏数据拦截 | `calories=-1`、JSON 解析失败不入库（BR-03） |
| 非法 diet_label | PUT /api/diet/profile 传入不在白名单的值返回 400 |
| 过敏原 100% 覆盖 | 含过敏原的餐厅必须在前端显示警告（自动化断言） |
| Session 隔离 | 两个浏览器 Session 互不影响 |
