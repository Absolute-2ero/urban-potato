## 1. 当前部署拓扑

```
单机部署（Linux Server / 本地开发机）

┌─────────────────────────────────────────────────────────┐
│  进程：uvicorn (FastAPI)         :8000  HTTP/HTTPS       │
│  进程：nginx（静态文件服务）      :80 / :443             │
│  进程：PostgreSQL 16             :5432                   │
│  进程：Elasticsearch 8.x         :9200                   │
│  进程：Redis 7                   :6379                   │
│  文件：data/food_db.sqlite        ./data/               │
│  文件：backend/.env（chmod 600）  ./backend/             │
└─────────────────────────────────────────────────────────┘
```

---

## 2. 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| 操作系统 | Ubuntu 22.04 LTS | 推荐；Debian 11+ 亦可 |
| Python | 3.11+ | 后端运行时 |
| Node.js | 18+ | 前端构建 |
| PostgreSQL | 16+ | 用户数据主库 |
| Elasticsearch | 8.x | 需安装 IK 中文分词插件 |
| Redis | 7+ | 缓存服务 |
| nginx | 1.24+ | 静态文件 + 反向代理 |
| Playwright Chromium | 自动下载 | 大众点评爬虫用 |
| 磁盘空间（最低） | 20 GB | ES 索引 + PG 数据 + 日志 |
| 内存（最低） | 4 GB | ES 默认 JVM heap 1GB |

## 4. 部署步骤

### 4.1 依赖安装

```bash
# Python 环境
python3.11 -m venv /opt/dietsearch/venv
source /opt/dietsearch/venv/bin/activate
pip install -r backend/requirements.txt

# Playwright 浏览器
playwright install chromium
playwright install-deps chromium

# 前端构建
cd frontend
npm ci
npm run build
```

### 4.2 Elasticsearch 安装与配置

```bash
# 安装 ES（Ubuntu）
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.x.x-amd64.deb
dpkg -i elasticsearch-8.x.x-amd64.deb

# 安装 IK 中文分词插件（必须，否则中文检索无法正常分词）
/usr/share/elasticsearch/bin/elasticsearch-plugin install \
    https://release.infinilabs.com/analysis-ik/stable/elasticsearch-analysis-ik-8.x.x.zip

# 复制同义词文件到 ES 配置目录
cp backend/data/diet_synonyms.txt \
    /etc/elasticsearch/analysis/diet_synonyms.txt

# 启动
systemctl enable elasticsearch
systemctl start elasticsearch

# 验证
curl http://localhost:9200/_cluster/health
```

### 4.3 数据库初始化

```bash
# PostgreSQL
sudo -u postgres psql <<EOF
CREATE USER dietsearch WITH PASSWORD 'your_password';
CREATE DATABASE dietsearch OWNER dietsearch;
\c dietsearch
EOF

# 运行 Schema 迁移
cd backend
python -m db.migrate   # 执行 schema.sql 创建表结构
```

```sql
-- schema.sql（节选）
CREATE TABLE users (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(64) UNIQUE NOT NULL,
    password_hash   VARCHAR(256) NOT NULL,
    email           VARCHAR(128) UNIQUE,
    created_at      TIMESTAMPTZ DEFAULT now(),
    last_login_at   TIMESTAMPTZ
);

CREATE TABLE diet_profiles (
    user_id         INT PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE,
    diet_labels     TEXT[] DEFAULT '{}',
    allergens       TEXT[] DEFAULT '{}',
    calorie_goal    INT,
    protein_goal_g  FLOAT,
    fat_goal_g      FLOAT,
    carb_goal_g     FLOAT,
    price_pref      SMALLINT,
    cuisine_prefs   TEXT[] DEFAULT '{}',
    updated_at      TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE food_log_entries (
    id              SERIAL PRIMARY KEY,
    user_id         INT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    log_date        DATE NOT NULL,
    meal_type       VARCHAR(16) NOT NULL,
    food_id         INT NOT NULL,
    food_name       VARCHAR(128) NOT NULL,
    quantity_g      FLOAT NOT NULL,
    calories_kcal   FLOAT NOT NULL,
    protein_g       FLOAT,
    fat_g           FLOAT,
    carb_g          FLOAT,
    created_at      TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_food_log_user_date ON food_log_entries(user_id, log_date);

CREATE TABLE saved_restaurants (
    id              SERIAL PRIMARY KEY,
    user_id         INT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    restaurant_id   VARCHAR(64) NOT NULL,
    restaurant_name VARCHAR(256) NOT NULL,
    saved_at        TIMESTAMPTZ DEFAULT now(),
    UNIQUE(user_id, restaurant_id)
);

CREATE TABLE search_feedbacks (
    id              SERIAL PRIMARY KEY,
    user_id         INT REFERENCES users(id),
    query_text      TEXT NOT NULL,
    restaurant_id   VARCHAR(64) NOT NULL,
    restaurant_name VARCHAR(256),
    rank_position   SMALLINT,
    is_relevant     BOOLEAN NOT NULL,
    created_at      TIMESTAMPTZ DEFAULT now()
);
```

### 4.4 Elasticsearch 索引初始化

```bash
cd backend
python -m ir.index_builder --create-index   # 创建索引（定义 Mapping）
python -m crawler.pipeline --city 北京 --source gaode --limit 500  # 初始数据
```

### 4.5 环境变量配置

```bash
# backend/.env（chmod 600）
POSTGRES_DSN=postgresql+asyncpg://dietsearch:your_password@localhost/dietsearch
REDIS_URL=redis://localhost:6379/0
ELASTICSEARCH_URL=http://localhost:9200

DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxx
GAODE_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxx

SESSION_SECRET=生成一个足够随机的字符串_至少32位

LLM_MODEL=deepseek-chat
ES_INDEX_NAME=restaurants
```

生成随机 Session Secret：
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

### 4.6 nginx 配置

```nginx
# /etc/nginx/sites-available/dietsearch
server {
    listen 80;
    server_name your_domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name your_domain.com;

    ssl_certificate     /etc/letsencrypt/live/your_domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your_domain.com/privkey.pem;

    # 前端静态文件
    root /opt/dietsearch/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;   # SPA fallback
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cookie_flags ~ httponly samesite=lax;
    }
}
```

### 4.7 systemd 服务

```ini
# /etc/systemd/system/dietsearch.service
[Unit]
Description=DietSearch FastAPI Backend
After=network.target postgresql.service elasticsearch.service redis.service

[Service]
User=www-data
WorkingDirectory=/opt/dietsearch/backend
EnvironmentFile=/opt/dietsearch/backend/.env
ExecStart=/opt/dietsearch/venv/bin/uvicorn main:app \
    --host 127.0.0.1 \
    --port 8000 \
    --workers 4 \
    --access-log \
    --log-level info
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable dietsearch
systemctl start dietsearch
systemctl status dietsearch
```

## 5. 本地开发启动

```bash
# 启动所有依赖服务（使用 Docker Compose 简化）
docker compose up -d postgres elasticsearch redis

# 后端
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # 填写本地配置
uvicorn main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev   # 访问 http://localhost:3000
```

`docker-compose.yml` 节选：
```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_USER: dietsearch
      POSTGRES_PASSWORD: dev_password
      POSTGRES_DB: dietsearch
    ports: ["5432:5432"]

  elasticsearch:
    image: elasticsearch:8.13.0
    environment:
      - discovery.type=single-node
      - ES_JAVA_OPTS=-Xms512m -Xmx512m
      - xpack.security.enabled=false
    ports: ["9200:9200"]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
```

## 6. 常见运维操作

```bash
# 查看后端日志
journalctl -u dietsearch -f

# 重建 ES 索引（数据不变，重建 Mapping）
python -m ir.index_builder --rebuild

# 手动触发城市爬取
python -m crawler.pipeline --city 上海 --source all --limit 300

# 备份 PostgreSQL
pg_dump -U dietsearch dietsearch > backup_$(date +%Y%m%d).sql

# 备份食物数据库（SQLite）
cp data/food_db.sqlite backups/food_db_$(date +%Y%m%d).sqlite

# 重启后端服务
systemctl restart dietsearch
```

## 7. 数据备份策略

| 数据 | 备份频率 | 存储位置 | 保留周期 |
|------|---------|---------|---------|
| PostgreSQL 全量 | 每日凌晨 2:00 | 本机 + 远端存储 | 30 天 |
| food_db.sqlite | 每次有新食物入库时 | 本机 | 10 个版本 |
| ES 索引 | 每周快照（Elasticsearch Snapshot API） | 本机 | 4 周 |
| `.env` | 首次配置后手动备份 | 加密离线存储 | 永久 |

```bash
# 自动备份 crontab
0 2 * * * pg_dump -U dietsearch dietsearch | gzip > /backups/pg_$(date +\%Y\%m\%d).sql.gz
```
