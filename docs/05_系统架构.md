## 4. 关键数据流

### 4.1 食物搜索（含 LLM fallback）

```
GET /api/food/search?q=红烧肉
    → food.router → FoodService.search(query)
        → SQLite FTS5 全文查询
        ├─ 命中 → 返回结果列表
        └─ 未命中 →
            → Redis GET food:llm:红烧肉
            ├─ 缓存命中 → 返回缓存
            └─ 未命中 →
                → LLM API（DeepSeek）
                → Redis SET food:llm:红烧肉 EX=604800
                → 返回 {data, requires_confirm: true}
                （前端显示确认弹窗，用户确认后 POST /api/food/confirm）

POST /api/food/confirm {food_data, food_name}
    → FoodService.confirm_and_store(food_data)
        → 检查 SQLite 唯一性（BR-04）
        → INSERT food_items(verified=0)
        → UPDATE SET verified=1
        → 返回 {food_id}
```

### 4.2 餐厅搜索

```
GET /api/search?q=素食&diet_labels=vegan&sort=default&lat=39.9&lng=116.4
    → search.router → SearchService.search(params)
        → QueryParser.parse(q)          # 分词、意图、同义词扩展
        → RankingService.get_weights(sort_mode)  # 读取 ranking.yaml
        → ES async_search(bool_query)
        → hits → RankingService.rerank(hits, user_profile)
                    → calc_diet_match_score(restaurant, query_labels, user_allergens)
                    → calc_distance_score(restaurant.geo, user_geo)
                    → weighted_sum → sorted
        → format_response(sorted_hits)   # 附加 highlights, allergen_warning
        → 返回 JSON
```

### 4.3 爬虫流水线（离线批处理）

```
python -m crawler.pipeline --city 北京 --source all

→ GaodeCrawler.fetch_all()
    → 高德 API 分页获取 POI（每页 20 条，总计 N 家）
    → 输出 List[RestaurantDoc]（骨架）

→ DianpingCrawler.enrich(restaurants)
    → 对每家餐厅 Playwright 访问大众点评页面
    → 提取菜品、标签、评分 → merge 到 RestaurantDoc

→ MeituanCrawler.enrich(restaurants)
    → httpx 请求美团，提取菜品价格 → merge

→ NLPLabeler.infer_diet_labels(restaurants)
    → 规则匹配 + 关键词分析 → 填充 diet_labels

→ Pipeline.dedup(restaurants)
    → 坐标近邻去重（R-tree 空间索引）

→ IndexService.bulk_index(restaurants)
    → ES bulk API，每批 100 条
    → 记录爬取日志
```

## 5. IR 核心设计

### 5.1 索引构建策略

```
初始构建（全量）：
  python -m ir.index_builder --source data/restaurants.json

增量更新（v1.2）：
  python -m ir.index_builder --mode incremental --since 2026-05-01
  → 仅重索引 indexed_at < since 或 source_updated_at > since 的文档
```

### 5.2 分析器配置（Elasticsearch）

```json
{
  "settings": {
    "analysis": {
      "analyzer": {
        "zh_diet_analyzer": {
          "type": "custom",
          "tokenizer": "ik_smart",
          "filter": ["lowercase", "zh_stop", "diet_synonyms"]
        },
        "en_diet_analyzer": {
          "type": "custom",
          "tokenizer": "standard",
          "filter": ["lowercase", "en_stop", "diet_synonyms", "porter_stem"]
        }
      }
    }
  }
}
```

### 5.3 查询构建示例

用户输入 `"gluten free high protein near me"`：

```json
{
  "query": {
    "bool": {
      "must": [
        {
          "multi_match": {
            "query": "gluten-free high-protein",
            "fields": ["name^3", "description", "diet_labels_text^2", "menu_items.name^1.5"],
            "type": "best_fields",
            "analyzer": "en_diet_analyzer"
          }
        }
      ],
      "filter": [
        { "terms": { "diet_labels": ["gluten-free", "high-protein"] } }
      ],
      "should": [
        {
          "function_score": {
            "gauss": {
              "geo": {
                "origin": "39.9,116.4",
                "scale": "2km",
                "offset": "500m",
                "decay": 0.5
              }
            }
          }
        }
      ]
    }
  },
  "highlight": {
    "fields": { "name": {}, "description": {}, "menu_items.name": {} }
  }
}
```

## 6. 安全设计

| 关注点 | 实现方式 |
|--------|----------|
| 传输安全 | 生产环境全站 HTTPS（Let's Encrypt 证书） |
| 身份鉴权 | Cookie Session（HttpOnly + Secure + SameSite=Lax） |
| 密码存储 | bcrypt 哈希，cost=12，不存明文 |
| API Key 管理 | 通过 `.env` + 环境变量注入，代码零硬编码 |
| 数据隔离 | 所有用户数据查询带 `WHERE user_id = ?`（BR-02） |
| 过敏原警告 | 后端计算并在响应中标注，前端不可绕过（BR-08） |
| 爬虫合规 | 遵守 robots.txt，设置合理请求间隔，不存储用户隐私数据 |
| LLM 输出过滤 | 解析 LLM 返回 JSON 前严格 Schema 校验，不信任原始字符串 |
