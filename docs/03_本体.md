# 数据模型（本体）

**版本：** v1.0
**更新日期：** 2026-05-22

---

## 1. 数据库概览

| 数据库 | 类型 | 用途 |
|--------|------|------|
| PostgreSQL | 关系型 | 用户账户、饮食档案、饮食日志、搜索反馈、收藏 |
| SQLite | 嵌入式文件 | 食物静态数据库 + LLM 扩展库（随代码分发，无需独立服务） |
| Elasticsearch 8.x | 文档索引 | 餐厅主索引、菜品子索引、全文检索与 Faceted 搜索 |
| Redis | 内存缓存 | Autocomplete 缓存、LLM 查询结果缓存、热门搜索词 |

---

## 2. 实体关系总览

```
users ─┬─► diet_profiles (user_id)         # 饮食档案（1对1）
       ├─► food_log_entries (user_id)       # 每日饮食记录（1对多）
       ├─► saved_restaurants (user_id)      # 收藏餐厅（多对多）
       └─► search_feedbacks (user_id)       # 搜索相关性反馈（1对多）

food_log_entries ──► food_items (food_id)  # 日志条目引用食物

food_items (SQLite)
  ├── source: "static"    # 预置静态库
  └── source: "llm_inferred"  # LLM 查询后入库

restaurants (Elasticsearch 索引)
  └── menu_items[]        # 嵌套文档（菜品）

search_feedbacks ──► restaurants (restaurant_id, 冗余字段)
```

## 3. PostgreSQL 表结构

### 3.1 users（用户）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 用户 ID |
| username | VARCHAR(64) UNIQUE NOT NULL | 用户名 |
| password_hash | VARCHAR(256) NOT NULL | bcrypt 哈希密码 |
| email | VARCHAR(128) UNIQUE | 邮箱 |
| created_at | TIMESTAMPTZ DEFAULT now() | 注册时间 |
| last_login_at | TIMESTAMPTZ | 最后登录时间 |

---

### 3.2 diet_profiles（饮食档案）

每个用户至多一条，注册时自动创建空档案。

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | INT PK FK→users.id | 用户 ID（1对1） |
| diet_labels | TEXT[] | 饮食类型标签（如 `{vegan, gluten-free}`） |
| allergens | TEXT[] | 过敏原（如 `{peanut, shellfish}`） |
| calorie_goal | INT | 每日热量目标（kcal），NULL 表示未设置 |
| protein_goal_g | FLOAT | 每日蛋白质目标（g） |
| fat_goal_g | FLOAT | 每日脂肪目标（g） |
| carb_goal_g | FLOAT | 每日碳水目标（g） |
| price_pref | SMALLINT | 价格偏好（1-4，NULL=不限） |
| cuisine_prefs | TEXT[] | 偏好菜系 |
| updated_at | TIMESTAMPTZ | 最后修改时间 |

**约束：**
- `diet_labels` 中的值必须属于预定义分类体系（应用层校验）
- `calorie_goal` > 0（若非 NULL）

---

### 3.3 food_log_entries（饮食日志条目）

记录用户每天摄入的食物，是饮食管理的核心表。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 条目 ID |
| user_id | INT FK→users.id NOT NULL | 所属用户 |
| log_date | DATE NOT NULL | 记录日期 |
| meal_type | VARCHAR(16) | 餐次：breakfast / lunch / dinner / snack |
| food_id | INT NOT NULL | 关联的食物 ID（来自 SQLite food_items） |
| food_name | VARCHAR(128) NOT NULL | 冗余食物名（防 food_items 变更影响历史） |
| quantity_g | FLOAT NOT NULL | 食用量（克） |
| calories_kcal | FLOAT NOT NULL | 本次摄入热量（= food.calories_per_100g × quantity / 100） |
| protein_g | FLOAT | 蛋白质（克） |
| fat_g | FLOAT | 脂肪（克） |
| carb_g | FLOAT | 碳水（克） |
| created_at | TIMESTAMPTZ DEFAULT now() | 记录时间 |

**索引：** `(user_id, log_date)` 联合索引（高频查询）

---

### 3.4 saved_restaurants（收藏餐厅）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| user_id | INT FK→users.id NOT NULL | |
| restaurant_id | VARCHAR(64) NOT NULL | Elasticsearch 文档 ID |
| restaurant_name | VARCHAR(256) NOT NULL | 冗余餐厅名（快速展示用） |
| saved_at | TIMESTAMPTZ DEFAULT now() | 收藏时间 |

**约束：** `(user_id, restaurant_id)` 唯一

### 3.5 search_feedbacks（搜索反馈）

记录用户对搜索结果的相关性反馈，用于离线 IR 评估。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | |
| user_id | INT FK→users.id | NULL 表示匿名用户 |
| query_text | TEXT NOT NULL | 原始查询词 |
| restaurant_id | VARCHAR(64) NOT NULL | 被反馈的餐厅 |
| restaurant_name | VARCHAR(256) | 冗余字段 |
| rank_position | SMALLINT | 该餐厅在结果列表中的位置（从 1 开始） |
| is_relevant | BOOLEAN NOT NULL | true=相关 / false=不相关 |
| created_at | TIMESTAMPTZ DEFAULT now() | |

## 4. SQLite 表结构（食物数据库）

### 4.1 food_items（食物条目）

文件路径：`data/food_db.sqlite`，随项目代码分发。

```sql
CREATE TABLE food_items (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    name_zh      TEXT NOT NULL,               -- 中文名称
    name_en      TEXT,                        -- 英文名称（可选）
    name_pinyin  TEXT,                        -- 拼音（用于拼音搜索）
    calories     REAL NOT NULL,              -- 每 100g 热量（kcal）
    protein_g    REAL NOT NULL DEFAULT 0,    -- 每 100g 蛋白质（g）
    fat_g        REAL NOT NULL DEFAULT 0,    -- 每 100g 脂肪（g）
    carb_g       REAL NOT NULL DEFAULT 0,    -- 每 100g 碳水（g）
    sodium_mg    REAL DEFAULT NULL,          -- 每 100g 钠（mg），可选
    fiber_g      REAL DEFAULT NULL,          -- 每 100g 膳食纤维（g），可选
    diet_labels  TEXT DEFAULT '[]',          -- JSON 数组，如 '["vegan","gluten-free"]'
    source       TEXT NOT NULL DEFAULT 'static',  -- static | llm_inferred | user_added
    verified     INTEGER NOT NULL DEFAULT 1, -- 1=已验证 / 0=待验证（LLM 数据初始为0）
    created_at   TEXT DEFAULT (datetime('now'))
);

-- 全文搜索索引
CREATE VIRTUAL TABLE food_fts USING fts5(
    name_zh, name_en, name_pinyin,
    content='food_items', content_rowid='id'
);
```

**数据来源说明：**
- `source='static'`：预置库，≥500 条，人工整理，数据可信
- `source='llm_inferred'`：LLM 查询后用户确认入库，`verified` 初始为 0，用户确认后改为 1
- `source='user_added'`：用户手动填写（v1.2 功能）

## 6. 数据流说明

### 6.1 食物搜索与日志记录

```
用户输入食物名
    → 查询 food_fts（SQLite FTS5 全文索引）
    ├─ 命中 → 返回 food_items 列表
    └─ 未命中 → 调用 LLM → 用户确认 → INSERT food_items(source='llm_inferred')
                                              → 返回新条目

用户选择食物 + 输入分量(g)
    → 计算热量 = calories × quantity / 100
    → INSERT food_log_entries
```

### 6.2 餐厅数据入库（爬虫 → ES）

```
Crawler 采集原始数据（JSON）
    → 数据清洗 & 标准化（字段映射到 Restaurant 文档结构）
    → NLP 饮食标签推断（基于菜品名和描述）
    → 去重（同名 + 坐标近邻）
    → Elasticsearch bulk index
```

### 6.3 搜索请求流

```
用户提交搜索
    → QueryParser（分词 + 意图识别 + 同义词扩展）
    → 构建 ES Bool Query（must + filter + should）
    → ES 返回 hits（含 _score + highlights）
    → 多维度重排序（BM25 + diet_match + rating + distance）
    → 格式化 → 返回前端
```
